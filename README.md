# Лабораторна робота №6 — CI/CD для автоматизації збору метрик, статичного аналізу коду і код-рев'ю

## Мета
Автоматизувати процес перевірки коду перед його інтеграцією в основну гілку репозиторію.

## Структура проєкту
- `src/` — вихідний код проєкту (модуль онлайн-календаря подій)
- `tests/` — модульні тести
- `.github/workflows/` — конфігурація CI/CD пайплайну
- `sonar-project.properties` — налаштування SonarQube аналізу

## Основні висновки
- Налаштовано CI/CD пайплайн через GitHub Actions з двома jobs: build та analysis.
- Автоматизовано запуск тестів (pytest), збір метрик коду (cloc), статичний аналіз (SonarQube).
- Інтегровано перевірку лінтером flake8 та форматером Black через Reviewdog з автоматичними коментарями до PR.
- Налаштовано обов'язкову перевірку наявності схваленого код-рев'ю перед злиттям у гілку develop.
- Branch protection rules гарантують, що неперевірений код не потрапить у головну гілку.
